# novel-writer-cn 功能拆解报告

## 基本信息
- **类型**：Claude Code Skill
- **版本**：1.0.0
- **触发条件**：用户要求写小说、创建故事剧情、设计角色、或开发小说叙事框架
- **适用场景**：言情/悬疑/玄幻/都市/科幻/武侠/历史/恐怖/校园/军事等全类型
- **核心机制**：三幕式剧情框架 + 4版本结局 + 关系矩阵

## 功能维度分析

### 1. 世界观构建
- **覆盖度**：❌ 不涉及
- **实现方式**：仅在"第一幕-铺垫"中提及"世界观设定"，但无具体构建方法
- **独特亮点**：无

### 2. 人物设定
- **覆盖度**：✅ 核心功能
- **实现方式**：三层角色体系——主角（姓名/年龄/外貌/性格/背景/动机/成长弧线）、核心配角（盟友/导师/情报提供者，各自有独特功能）、反派（动机/背景/对立点）；定制化选项（主角数量/情感基调/结局偏好/篇幅/特定元素）
- **独特亮点**：强调每个配角的"独特功能"——防止工具人化；反派必须有独立动机和对立点设计

### 3. 情节设计
- **覆盖度**：✅ 核心功能
- **实现方式**：完整三幕式结构——第一幕（世界观/日常/激励事件/拒绝改变）、第二幕（新挑战/中点转折/反派逼近/一切尽失低谷）、第三幕（最终准备/高潮对决/结局呈现）；关键剧情节点标注
- **独特亮点**：三幕式是好莱坞经典叙事框架，适合传统出版向小说；"一切尽失"低谷点是市面上唯一明确标注此节点的 skill

### 4. 章节创作
- **覆盖度**：❌ 不涉及
- **实现方式**：仅输出创作框架，不涉及逐章正文生成
- **独特亮点**：无

### 5. 一致性审查
- **覆盖度**：❌ 不涉及
- **实现方式**：无
- **独特亮点**：无

### 6. 文风控制
- **覆盖度**：⚡ 部分覆盖
- **实现方式**：用户可指定情感基调（轻松幽默/深沉虐心/热血激昂）和小说风格
- **独特亮点**：支持10种小说类型的定制化建议

### 7. 平台适配
- **覆盖度**：❌ 不涉及
- **实现方式**：无
- **独特亮点**：无

### 8. 协作与迭代
- **覆盖度**：⚡ 部分覆盖
- **实现方式**：框架可随用户反馈调整；支持定制化选项让用户在生成前指定偏好
- **独特亮点**：多版本结局设计（4种：经典圆满/悲剧/开放/反转）本身就是迭代思维的体现

## 总结
- **核心优势**：人物设定三层体系最结构化；多版本结局是差异化能力；三幕式结构适合出版向；10种小说类型的广泛覆盖
- **主要局限**：仅输出框架而非正文；无审查/伏笔/记忆系统；无平台适配；无自动化脚本
- **最佳适用场景**：小说创作前期的框架搭建；需要多版本结局探索的故事设计；作为 Advisor 角色的输入参考
- **在双终端工作流中的建议定位**：Terminal A 策划端 —— 人物设定框架参考 + 多版本结局探索
