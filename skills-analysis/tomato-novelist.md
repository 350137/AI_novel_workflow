# tomato-novelist 功能拆解报告

## 基本信息
- **类型**：Claude Code Skill
- **版本**：1.0.0（推测）
- **触发条件**：创作中文小说/分章节故事/长篇小说/番茄小说/网文写作
- **平台定位**：番茄小说平台特调版（基于番茄平台实战数据优化）
- **核心机制**：3阶段创作流程（6问确认→规划+二次确认→疯狂创作）+ 情绪过山车循环 + 每5章数据反馈环

## 功能维度分析

### 1. 世界观构建
- **覆盖度**：❌ 不涉及
- **实现方式**：仅在题材选择中间接涉及世界观元素；无独立世界观构建模块
- **独特亮点**：无

### 2. 人物设定
- **覆盖度**：✅ 核心功能
- **实现方式**：问题3（主角设定）+问题4（核心反差点）——6种反差类型模板（表面摆烂实际卷王/社恐但装逼/被虐后觉醒反杀/老实人黑化/隐藏大佬装萌新/废柴实则天赋异禀）；01-人物档案.md（character-template-tomato.md 模板）
- **独特亮点**：反差点设计是番茄爆款核心——直接对应推荐算法；6种反差类型覆盖主流爽文人设

### 3. 情节设计
- **覆盖度**：✅ 核心功能
- **实现方式**：问题5（核心冲突——7种类型：生死存亡/查明真相/爱情阻碍/复仇雪恨/权力争夺/成长突破/守护保护）；问题6（章节数量规划）；00-大纲.md（outline-template-tomato.md）；02-情绪曲线.md（tomato-emotion-curve.md——情绪过山车设计）；每3章"压-小扬-压-爆"循环
- **独特亮点**：情绪标签（问题1）直接决定番茄算法推荐——6种标签覆盖全平台爆款类型；情绪过山车是番茄平台的差异化设计

### 4. 章节创作
- **覆盖度**：✅ 核心功能
- **实现方式**：逐章创作4步骤——写前分析（读取大纲/上章钩子/情绪曲线/设计开头钩子/规划3-5场景）→撰写（chapter-template.md模板/2200-2800字/对话规范/内容扩充技巧/结尾钩子10种类型/初稿字数粗检）→撰写后优化（连贯性检查/深度润色去AI味加番茄味/质量自查>60分/字数脚本检查）→最后收尾（生成摘要/更新状态/记录钩子使用）
- **独特亮点**：开篇前50字强制要求抓人（书架页展示空间）；10种钩子类型轮换使用；每5章生成节奏报告+询问数据反馈——唯一将平台数据指标纳入创作流程的 skill

### 5. 一致性审查
- **覆盖度**：⚡ 部分覆盖
- **实现方式**：连贯性检查（consistency.md）；质量自查（quality-checklist.md，分数>60交付）；字数检查脚本（check_chapter_wordcount_tomato.py）
- **独特亮点**：质量自查有量化分数（>60分交付）；字数检查用脚本而非 LLM 估算

### 6. 文风控制
- **覆盖度**：✅ 核心功能
- **实现方式**：深度润色（去AI味+加番茄味）——去除过度修饰形容词/减少抽象陈述改具体动作/避免四字格律堆砌/增加口语化/情绪高潮处加入可划线评论金句/长短句交替优化节奏；黄金开篇03-黄金开篇.md（3版本开篇50字供选择）
- **独特亮点**：可划线评论金句是番茄平台特有交互（读者长按划线评论）；黄金开篇3版本提供A/B测试可能；番茄味润色（口语化+金句）与通用去AI味是两个层次

### 7. 平台适配
- **覆盖度**：✅ 核心功能（最强项）
- **实现方式**：番茄情绪标签（决定算法推荐）；2200-2800字最优阅读长度（基于番茄平台数据）；开篇前50字书架页展示优化；每5章询问完读率/追读率/评论关键词并据此调整；章节标题含钩子设计；评论区诱导设计
- **独特亮点**：唯一明确为特定平台优化的 skill；情绪标签→算法推荐的映射是实战数据沉淀；数据反馈环（完读率/追读率→调整大纲）是数据驱动创作

### 8. 协作与迭代
- **覆盖度**：✅ 核心功能
- **实现方式**：6问确认（逐一询问等待回复，不做假设）；规划二次确认（展示摘要→等用户确认→才进入创作）；每章完成后询问是否继续；每5章数据反馈→调整后续大纲（为高人气配角加戏/强化爆款情绪标签）；TODO list 管理章节进度
- **独特亮点**：每步都有确认点（6问→规划确认→每章确认→每5章反馈）——人机协作节奏最尊重人类决策权；数据反馈驱动的迭代（不是凭感觉调整）

## 总结
- **核心优势**：平台适配能力是9个 skills 中最强的（番茄小说专属优化）；数据反馈驱动创作（完读率/追读率→调整）；情绪标签体系对应推荐算法；10种钩子类型轮换；人机协作节奏设计最尊重人类决策权
- **主要局限**：仅适配番茄平台；无世界观构建模块；无本地自动化脚本（除字数检查外）；无伏笔生命周期管理；依赖人类提供数据反馈
- **最佳适用场景**：番茄小说平台的连载创作；需要数据驱动迭代的网文项目；需要严格的平台规范（字数/情绪/钩子/开篇）的场景
- **在双终端工作流中的建议定位**：Terminal B 写作端 —— 番茄平台特化的 Writer（平台规范+数据反馈环+钩子管理）
