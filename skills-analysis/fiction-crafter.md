# fiction-crafter 功能拆解报告

## 基本信息
- **类型**：Claude Code Skill
- **版本**：1.1.0
- **作者**：kimo（基于 itjhg/novel-generator 二次开发）
- **触发条件**：用户提供小说方向/题材/关键词，或提到"爽文/小说/写作"
- **适用场景**：都市、修仙、玄幻、重生、系统流等网文题材
- **核心机制**：`.learnings/` 记忆系统 + 7步工作流（提示词生成→大纲→卷规划→逐章→格式化→审查→图解）

## 功能维度分析

### 1. 世界观构建
- **覆盖度**：⚡ 部分覆盖
- **实现方式**：在提示词生成阶段自动补全"力量体系、社会规则、时代背景"；力量体系要求有明确层级；通过 STORY_BIBLE.md 持续沉淀世界观规则
- **独特亮点**：强制要求力量体系层级清晰（便于体现碾压感），世界观规则变更时持续记录到 STORY_BIBLE.md

### 2. 人物设定
- **覆盖度**：⚡ 部分覆盖
- **实现方式**：提示词阶段定义主角人设（初始身份、性格、金手指）、配角框架（对手/盟友/红颜各至少1人）；CHARACTERS.md 持续追踪角色状态变化（升级、受伤、死亡）；配角出场后>15章未出现触发预警
- **独特亮点**：角色成长曲线追踪（连续5章无成长=预警）；配角命运追踪机制防止"人间蒸发"

### 3. 情节设计
- **覆盖度**：✅ 核心功能
- **实现方式**：全局大纲→卷级章节规划（树状分解：卷→章节核心任务/爽点类型/关键情节/情绪曲线）→逐章生成；起承转合设计；关键转折点标注
- **独特亮点**：卷级章节规划是"防止剧情写偏的关键步骤"；每章标注情绪曲线类型（低开高走/层层递进/反转爆发）

### 4. 章节创作
- **覆盖度**：✅ 核心功能
- **实现方式**：按固定模板生成（章名→概要→爽点→情绪曲线→正文→章末钩子）；每章2000-3000字；生成前必读6个记忆文件；爽文节奏公式（每1-2章小打脸，每3-5章中打脸，每8-12章大高潮）
- **独特亮点**：爽文节奏公式化为具体章数间隔；强制章末钩子；生成前必须读取全部记忆文件（防穿帮）

### 5. 一致性审查
- **覆盖度**：✅ 核心功能
- **实现方式**：9维完整审查体系（核心任务达成/有效信息占比/穿帮检测/角色成长追踪/套路重复检测/爽点结构完整性/伏笔管理/承接与前文连贯/卷级目标进度）；2种模式（快速审3维/完整审9维）；穿帮检测覆盖4类（角色/地点/能力/时间线）；套路重复相似度对比（>80%高警，>60%中警）
- **独特亮点**：审查维度最丰富（9维）；按章节重要性选择审查深度；历史趋势追踪

### 6. 文风控制
- **覆盖度**：⚡ 部分覆盖
- **实现方式**：6条创作原则（强代入感/快节奏/层层递进/装逼打脸/金手指合理/伏笔呼应）；6条禁忌事项；对话要求有个性差异；format_chapter.py 统一章节分隔符格式
- **独特亮点**：禁忌事项清单明确；格式化脚本保证输出规范

### 7. 平台适配
- **覆盖度**：❌ 不涉及
- **实现方式**：无特定平台适配（通用网文格式）
- **独特亮点**：无

### 8. 协作与迭代
- **覆盖度**：⚡ 部分覆盖
- **实现方式**：续写/中断恢复机制（读取8个文件→检查进度→确认任务→生成）；失败记录系统 ERRORS.md（6种失败类型+修正方案+预防措施）；提示词迭代（用户确认或调整）；init-novel.sh 项目初始化脚本
- **独特亮点**：失败记录系统可沉淀为创作模式（"捕获→记录→沉淀→复用"）；续写检查清单确保中断恢复质量

## 总结
- **核心优势**：9维审查体系最完整；卷级章节规划防止剧情偏移；失败记录系统支持持续改进；爽文节奏公式化
- **主要局限**：无平台特定优化；人物设定深度不如 novel-writer-cn；无本地 NLP 能力（纯 LLM 驱动）；记忆系统为 Markdown 文件（非结构化数据库）
- **最佳适用场景**：网文爽文长篇创作（修仙/都市/玄幻），需要严格的剧情一致性审查
- **在双终端工作流中的建议定位**：Terminal A 策划端 —— 创意生成+大纲规划+质量审查
