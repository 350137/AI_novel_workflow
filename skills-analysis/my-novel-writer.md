# my-novel-writer 功能拆解报告

## 基本信息
- **类型**：Claude Code Skill（含 Python 核心引擎）
- **版本**：1.0.0
- **依赖**：Python3 + OpenAI 兼容 API（NOVEL_API_KEY 必填）
- **触发条件**：通过 CLI 命令调用（`novel_writer --new/--set-character/--generate` 等）
- **核心机制**：Python 核心引擎（core.py + config_models.py + prompts.py + text_processor.py）+ JSON 记忆管理（novel_context.json）+ 模板资源包

## 功能维度分析

### 1. 世界观构建
- **覆盖度**：✅ 核心功能
- **实现方式**：`--set-world` 命令设定世界观；world_building.md 模板（覆盖：地理/历史/文化/政治/经济/科技/种族/魔法系统/势力分布）；world_building.md 结构化设定表
- **独特亮点**：设定表覆盖9个子维度；通过 pydantic 模型验证配置完整性

### 2. 人物设定
- **覆盖度**：✅ 核心功能
- **实现方式**：`--set-character` 命令添加角色；character_card.md 模板（7段：基本信息/外貌/性格/背景/能力/关系/成长弧光）；通过 novel_context.json 持续追踪角色状态
- **独特亮点**：模板化角色卡 + JSON 结构化存储；支持成长弧光设计

### 3. 情节设计
- **覆盖度**：⚡ 部分覆盖
- **实现方式**：`--add-outline` / `--update-outline` 管理大纲；chapter_outline.md 模板；novel_structure_guide.md 参考指南（三幕式/人物弧光/伏笔技巧/节奏控制）；plot_tracker.py 伏笔追踪脚本
- **独特亮点**：大纲支持动态调整（`--update-outline`）；伏笔追踪脚本（记录开启/发展/关闭状态）

### 4. 章节创作
- **覆盖度**：✅ 核心功能
- **实现方式**：`--generate <章节号> <字数>` 命令生成章节；每章独立 md 文件；支持批量生成（bash for 循环）；自动生成本章摘要存入记忆库
- **独特亮点**：每次只生成一章避免上下文超限；支持自定义 API 端点和模型选择；生成后自动更新 JSON 记忆

### 5. 一致性审查
- **覆盖度**：⚡ 部分覆盖
- **实现方式**：5类生成时约束——字数硬指标（2200-2500字）/伏笔追踪/人物状态连贯（受伤须恢复）/战力体系逻辑（跨境需金手指或计谋支撑）/违禁词替换；plot_tracker.py 检查未关闭伏笔；QUALITY_CHECKLIST 质量检查清单
- **独特亮点**：字数硬校验（<2000字自动补全）；战斗结果逻辑约束（以弱胜强需≥1项支撑）

### 6. 文风控制
- **覆盖度**：✅ 核心功能
- **实现方式**：`--set-style` 命令锁定风格（如"古风武侠，语言半文半白"）；违禁词替换规则（杀→陨落、死→魂飞魄散、血→猩红）；第一人称强制锁定；爽点密度要求（每章≥1小爽点）；作者说结尾引导互动
- **独特亮点**：风格锁定+违禁词替换双重保障；作者说引导读者互动（求收藏/求票）；补全机制（心理独白/环境渲染/动作细节/配角反应）

### 7. 平台适配
- **覆盖度**：⚡ 部分覆盖
- **实现方式**：章节标题含钩子（"苏天陨落""玉坠觉醒"）吸引点击；作者说引导读者互动行为
- **独特亮点**：网文平台（起点/番茄风格）的标题钩子设计

### 8. 协作与迭代
- **覆盖度**：✅ 核心功能
- **实现方式**：`--status` 查看进度；`--update-outline` 动态调整大纲；`--new` 初始化新项目；完整 CLI 命令体系（7个核心命令）；JSON 格式记忆支持增量更新和多工具读取
- **独特亮点**：最完整的 CLI 命令体系；JSON 记忆格式最易于程序化处理；支持环境变量配置（多模型/多端点）

## 总结
- **核心优势**：唯一拥有完整 Python 核心引擎的 skill（非纯 prompt）；CLI 命令体系最完善；JSON 记忆格式最适合程序化处理和集成；字数硬校验和自动补全机制；伏笔追踪脚本可独立运行
- **主要局限**：依赖外部 API（需配置密钥）；生成质量取决于所选模型；无独立审计/审查 Agent 分离；记忆结构为单 JSON 文件（不够细粒度，750章规模可能不够用）
- **最佳适用场景**：需要程序化控制和 CLI 调用的小说项目；与其他工具（如 onkos）集成的基础设施层；批量章节生成
- **在双终端工作流中的建议定位**：Terminal B 写作端 —— Writer Agent 的生成引擎层（提供 `--generate` 能力 + 风格锁定 + 字数控制）
