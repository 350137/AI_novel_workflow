# story-cog 功能拆解报告

## 基本信息
- **类型**：Claude Code Skill
- **版本**：1.0.10
- **作者**：CellCog
- **依赖**：CellCog API（远程智能）+ Python3 + CELLCOG_API_KEY
- **触发条件**：AI 创意写作和故事叙述——小说、短篇、剧本、同人、诗歌
- **核心机制**：CellCog 远程 API 封装——本质上是一个"远程智能代理"，零本地推理能力

## 功能维度分析

### 1. 世界观构建
- **覆盖度**：⚡ 部分覆盖
- **实现方式**：通过 prompt 描述世界观需求，由远程 CellCog API 生成；提供世界构建示例 prompt（蒸汽朋克奇幻——地理/魔法系统/社会结构/历史事件/势力/科技水平/5个有趣地点）
- **独特亮点**：prompt 模板结构化程度高，覆盖7个世界构建子维度

### 2. 人物设定
- **覆盖度**：⚡ 部分覆盖
- **实现方式**：角色圣经生成（背景故事/心理学/关系/语言模式/行为举止/自我欺骗的真相/改变触发点）；提供详细的人物 prompt 模板
- **独特亮点**：角色心理学深度（恐惧/欲望/防御机制）是其他 skills 缺少的维度

### 3. 情节设计
- **覆盖度**：⚡ 部分覆盖
- **实现方式**：三幕式结构支持；大纲生成；情节漏洞修复；提供各类型的 prompt 示例
- **独特亮点**：支持"帮助我解决情节漏洞"的对话式情节开发

### 4. 章节创作
- **覆盖度**：✅ 核心功能
- **实现方式**：全类型创作——闪小说（500字）/短篇（3000字）/微型小说（100字）/选集作品；章 draft；剧本格式（电视剧/电影/短片）；同人创作（续写/AU/跨界/缺失场景）；支持 agent 模式和 agent team 模式
- **独特亮点**：是唯一支持剧本格式的 skill；agent team 模式适合复杂叙事；支持同人创作是差异化能力

### 5. 一致性审查
- **覆盖度**：❌ 不涉及
- **实现方式**：无——依赖远程 API 的生成能力，无独立审查
- **独特亮点**：无

### 6. 文风控制
- **覆盖度**：⚡ 部分覆盖
- **实现方式**：通过 prompt 指定语调（哲学性/苦乐参半/忧郁但充满希望）；6条写作建议（类型期望/角色驱动/具体细节/情感真实/展示不告知/迭代）
- **独特亮点**：对"情感真实"的强调——即使奇幻设定也要真实情感

### 7. 平台适配
- **覆盖度**：❌ 不涉及
- **实现方式**：无
- **独特亮点**：无

### 8. 协作与迭代
- **覆盖度**：⚡ 部分覆盖
- **实现方式**：支持 OpenClaw fire-and-forget 模式（异步）；支持多 Agent provider（openclaw/cursor/claude-code/codex）；迭代建议（"初稿只是起点"）
- **独特亮点**：支持跨 Agent 平台的远程调用；异步模式适合大批量生成

## 总结
- **核心优势**：唯一支持剧本格式；远程 API 不占用本地上下文；跨 Agent 平台兼容；同人创作支持；角色心理学深度
- **主要局限**：零本地推理（纯 API 封装）；无审查/记忆/伏笔管理；依赖外部 API 可用性和费用；无中文优化
- **最佳适用场景**：需要剧本格式输出的项目；英文小说创作；作为远程智能补充（当本地上下文不足时卸载复杂任务）
- **在双终端工作流中的建议定位**：辅助角色 —— 剧本格式输出 / 角色心理学深度分析 / 同人创作场景
