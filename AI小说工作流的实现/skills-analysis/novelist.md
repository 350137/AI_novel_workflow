# novelist 功能拆解报告

## 基本信息
- **类型**：Claude Code Skill
- **版本**：1.0.0
- **触发条件**：用户输入关键词自动路由——设定/人设/金手指/世界观→世界观与角色构筑；大纲/剧情/卡文→结构工程；写一段/起草/扩写→场景渲染；润色/精修/去AI味→文字淬炼
- **适用场景**：网文/类型小说创作，强调"白金级网文大神"定位
- **核心机制**：4工作区路由 + memory/{项目名}/ 文件记忆 + 强制句法红线

## 功能维度分析

### 1. 世界观构建
- **覆盖度**：✅ 核心功能
- **实现方式**：力量体系层级设计（每层有明确标志+突破代价+天花板控制）；核心矛盾三问（主角想要什么/真正需要什么/谁在阻止）；金手指必须有代价；存档至 memory/{项目名}/worldbuilding.md
- **独特亮点**：强制金手指代价（"否则是无趣的外挂"）；境界突破不是"经验值满了就升级"；天花板可控防止后期崩坏

### 2. 人物设定
- **覆盖度**：✅ 核心功能
- **实现方式**：角色卡包含6要素（姓名/表层身份/底层执念/性格缺陷/专属口头禅或行为习惯/人物关系）；强调"性格缺陷让角色真实"
- **独特亮点**：底层执念+性格缺陷的双层驱动设计；专属口头禅/行为习惯增加辨识度

### 3. 情节设计
- **覆盖度**：✅ 核心功能
- **实现方式**：双结构模型（爽文用"目标-障碍-代价-反转"，传统叙事用"起承转合"）；卷大纲→章纲细化（每章含核心冲突/角色互动目的/结尾钩子）；爽点节拍器（每3章小高潮→每10章大反转）；卡文急救（检查目标/对立面/提出3个走向）
- **独特亮点**：双结构模型适配不同类型；爽点节拍器精准控制节奏；卡文急救机制

### 4. 章节创作
- **覆盖度**：✅ 核心功能
- **实现方式**：场景渲染工作区——前置读取3个记忆文件→确认场景边界→推镜法则（微观感官开头→拉出宏观）→展示代替告知→单场景闭环（1000-1500字）→伏笔追踪卡→状态回写
- **独特亮点**：推镜法则（感官切入，绝不上帝视角全景）；展示代替告知的强制规则；单场景闭环防止上下文膨胀

### 5. 一致性审查
- **覆盖度**：❌ 不涉及
- **实现方式**：无独立审查模块——但伏笔追踪卡提供基础的连续性保障；前置读取机制间接防穿帮
- **独特亮点**：前置读取是隐式的一致性保护

### 6. 文风控制
- **覆盖度**：✅ 核心功能（文字淬炼工作区）
- **实现方式**：5条强制句法红线——白描为主（一页最多1个比喻）、肯定句写作（禁止否定描述）、短句与句号（禁用破折号拆解）、禁止上帝视角总结、中文语感（避免西式从句）；黑名单硬替换（9组词汇/标点）；否定句转肯定句；剥离心理锚点（删除"他知道/他感觉到/他意识到"）；对话排毒；形容词压缩（1名词最多1形容词）；逐句自检
- **独特亮点**：句法红线是市面上最严格的 AI 味清除规则；黑名单硬替换不做商量；逐句自检保证0违规输出

### 7. 平台适配
- **覆盖度**：❌ 不涉及
- **实现方式**：无
- **独特亮点**：无

### 8. 协作与迭代
- **覆盖度**：⚡ 部分覆盖
- **实现方式**：精修输出格式（精修前原文→精修后→修改说明）；4工作区之间通过 memory/ 文件传递状态
- **独特亮点**：修改说明列出"为什么这么改"——教育用户提升写作能力

## 总结
- **核心优势**：文风控制能力是9个 skills 中最强的（句法红线+黑名单+逐句自检）；推镜法则和展示代替告知是最专业的场景写作指导；金手指代价和境界突破有代价的设定哲学
- **主要局限**：无独立审查模块；无平台适配；无自动化脚本（纯 prompt 驱动）；无伏笔生命周期管理
- **最佳适用场景**：追求文学品质的类型小说；需要强力去 AI 味的精修场景；作为文字淬炼的终极过滤器
- **在双终端工作流中的建议定位**：Terminal B 写作端 —— 文字淬炼/润色官（Polisher）角色
