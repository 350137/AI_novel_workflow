# write-volume — 卷级批量写作

> 批量运行多章流水线。每5章暂停让人类抽查，卷末强制暂停输出卷末报告。

---

## 调用方式

```
/write-volume <卷号> --from <起始章>
/write-volume 1 --from 11         # 从第11章开始写卷1剩余章
/write-volume 1                    # 从卷首开始写整卷
```

---

## 执行流程

### Step 0：加载卷配置

读取 `novel_memory/state/workflow_state.json`。

确定起止章号：
- 如果指定 `--from` → 从该章开始
- 否则 → 从 `volume_progress.chapters_completed + 1` 开始
- 结束章 = 该卷最后一章（从 volume_map 读取）

检查卷的 `detailed_plan`：
- 如果正在开新卷且 `detailed_plan` 为 null → 先执行开卷详细规划（读取初始大纲 + 前文状态 + 人物现状 + 关系线现状 + 活跃伏笔 → 生成 `volume_N_detailed_plan.md`）

---

### Step 1：逐章循环

```
for chapter in start_chapter..end_chapter:
    
    1. 运行 STEP 1-6 完整流水线（同 /write-chapter 的逻辑）
    
    2. 如果 verdict = NEEDS_HUMAN:
       - 输出当前章节的问题
       - 暂停，等待人类决策（跳过/手动修复/重试）
    
    3. 如果 chapter % 5 == 0 且 chapter != end_chapter:
       - 暂停，输出最近5章摘要：
         * 每章标题 + 审计分数 + 修订次数
         * 伏笔状态变化
         * 人物状态变化
         * 关系线推进
       - 询问人类是否继续
       - 人类确认 → 继续下一章
    
    4. 如果 chapter == end_chapter（卷末）:
       - 强制暂停
       - 输出卷末报告（见下方）
```

---

- 本卷新埋：Y 条
- 本卷回收：Z 条

#### 人类审批

- 审批通过 → 更新 volume_progress，标记该卷为 done
- 如有问题 → 等待人类指令
