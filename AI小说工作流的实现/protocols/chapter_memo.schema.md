# ChapterMemo Schema v1

> Planner 为每章产出的写作指令。7段 YAML frontmatter + Markdown body。
> Writer 以此为核心写作指令，Auditor 以此对照检测备忘偏离。

---

## 完整格式

```yaml
---
chapter: 11                              # 章节编号
title: "门框上的证据"                      # 章节标题
goal: "把某地点被动过手脚从猜测钉成实证"     # ≤50字，一句话核心目标
rhythm: "蓄压"                            # 爆发 | 蓄压 | 后效 | 过渡
wordCount: "2200-2500"                    # 目标字数范围
pov: "第一人称（林云）"                    # 视角声明
mood: "专注→冷冽满足"                      # 情绪基调
chapterType: "蓄压章"                      # 爆发章 | 蓄压章 | 后效章 | 过渡章
isGoldenOpening: false                    # 仅全书第一章为true
threadRefs: ["H003", "H007"]              # 关联伏笔ID列表
krRef: "KR1"                              # 本章推进的KR编号
hookType: "悬念钩子"                       # 章末钩子类型（10种之一）
sensoryAnchor: "触觉——松针的刺痛、石阶的粗粝"  # 本章五感着力点
---

- [ ] <辅助任务1 — 支线/日常进展>
- [ ] <辅助任务2>

- [ ] <兑现项1 — 前几章积累的期待，本章必须满足>
- [ ] <兑现项2>

## 3. 本章扣留（Held-Back）
- <扣留项1 — 读者想知道的，本章故意不给。标注：读者在期待什么/为什么不给/预计第N+X章揭晓>

- <扣留原因>

## 4. 日常/过渡功能（Daily/Transition Function）
<!-- 非高压章：每段非冲突内容承担什么功能——世界观展示/人物关系深化/为后续伏笔铺路/展示时间流逝/感官沉浸 -->
<!-- 高压章（爆发章/卷末章）：写"不适用——本章全程高压" -->

## 5. 三问测试（3-Question Test）
1. 如果跳过本章，读者会错过什么？→
2. 本章的爽点/看点是什么？→
3. 本章推进了哪条主线/支线？→

## 6. 章末状态变更（End-of-Chapter State Changes）

### 人物状态变更
| 角色 | 维度 | 从 | 到 | 原因 |
|------|------|----|----|------|

### 伏笔活动
| 伏笔ID | 操作 | 本章具体做了什么 | 推进后状态 | 预计下次推进章 |
|--------|------|---------------|----------|-------------|

### 关系线变更
| 关系线ID | 角色对 | 从阶段 | 到阶段 | 触发事件 |
|---------|--------|--------|--------|---------|

### 本章新增信息
- <entity>.<attribute> = <value> （重要性：permanent | arc_scoped | chapter_scoped）

## 7. 本章禁止事项（Don'ts）
- <禁止1 — 来自 Genre Profile prohibitions>
- <禁止2 — 本章特有的约束>
- <禁止3 — 疲劳词限制：本章重点避免的词>
- <禁止4 — AI标记词限制>
- <禁止5 — OOC红线>
```

## 字段说明

### YAML Frontmatter（机器可解析）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| chapter | int | ✅ | 章节编号 |
| title | string | ✅ | 章节标题 |
| goal | string | ✅ | 核心目标，≤50字 |
| rhythm | enum | ✅ | 爆发/蓄压/后效/过渡 |
| wordCount | string | ✅ | 目标字数范围 |
| pov | string | ✅ | 视角声明 |
| mood | string | ✅ | 情绪基调 |
| chapterType | enum | ✅ | 章节类型 |
| isGoldenOpening | bool | ✅ | 仅第1章为true |
| threadRefs | array | | 关联伏笔ID |
| krRef | string | ✅ | 推进的KR编号 |
| hookType | string | ✅ | 章末钩子类型 |
| sensoryAnchor | string | ✅ | 五感着力点 |

### Body（7段）

| 段号 | 名称 | 功能 |
|------|------|------|
| 1 | 本章任务 | Writer 的 to-do list |
| 2 | 必须兑现 | 前章积累的读者期待，本章必须满足 |
| 3 | 本章扣留 | 故意不给的信息——何时揭晓 |
| 4 | 日常功能 | 非高压段的叙事功能说明 |
| 5 | 三问测试 | 本章不可替代性自证 |
| 6 | 状态变更 | 人物/伏笔/关系/事实 四项变更 |
| 7 | 禁止事项 | 硬约束清单 |

---

## Auditor 对照规则

Auditor 读取 ChapterMemo 后：
- 对照 §1 → Layer 2 任务达成率
- 对照 §2 → Layer 2 兑现检查
- 对照 §3 → Layer 2 扣留检查（如果写了扣留项→备忘偏离）
- 对照 §6 → Layer 2 状态矛盾检测
- 对照 §7 → Layer 1+3 违禁检查

---

## 稀疏 Memo 豁免

过渡章/后效章 Memo 内容自然较少。以下情况不扣分：
- §4（日常功能）可写"本章承担世界观展示功能"
- §2（必须兑现）可写"本章无强制性兑现"
- §1（任务列表）可只有1条核心任务 + 1条辅助任务
- 但 §5（三问测试）和 §7（禁止事项）不可省略
