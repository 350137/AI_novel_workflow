# 润色规则

> 来源：inkos Polisher + novelist（句法红线+黑名单+心理锚点剥离）+ openclaw（反AI写作v4）+ onkos（风格指纹匹配）
> 使用方：Polisher

---

## 一、Step 1 — 加载并分析输入

加载以下文件：
- `chapter_N_revised.md` — 润色对象
- `chapter_N_audit.json` — 了解被标记过的区域（重点关注这些段落的润色质量）
- `revision_notes.md` — 了解哪些段落被 Reviser 修改过
- `story/style/genre_profile.md` — 违禁词表/疲劳词表/禁止事项
- `story/roles/<角色>.md` — 确认本章涉及角色的状态描述不被润色改变（境界名/功法名/武器名/道具名/伤势描述——这些是"事实"不是"文笔"）

分析——标注四类段落：

**□ 需要重点润色的段落**：被 Auditor 标记过的段落（已被 Reviser 修改→确认修改后的文笔流畅）/ 对话占比高的段落（对话自然度是润色重点）/ 动作场景段落（检查句号使用/短句节奏）/ 环境描写段落（检查感官分布是否足够）

**□ 不可触碰的区域**：作者说段落 / 隐藏情节追踪卡 / 章末钩子的核心句 / 角色的口头禅和独特说话方式（是个性标签，不是语言缺陷） / 已被 Auditor 标记 CRITICAL 且 Reviser 已修过的段落（双重改动风险）

**□ 高峰体验句保护**：识别正文中情绪强度达到顶点的句子——战斗高潮时的爆发句、情感转折时的关键句、揭示真相时的揭示句、章末钩子的最后一句。这些句子标记为"PROTECTED"——只做最小标点/格式调整，不改措辞、不改节奏。理由：这些句子是读者会产生情感反应的句子——过度润色会冷却其冲击力。

## 二、Step 2 — 执行润色（temp=0.5，逐段进行）

### Priority 1：AI腔移除（最高优先级——必须执行）

**1. 套话模式**（来源：openclaw anti-ai-v4）：
- "在……的过程中" → 改写为具体动作
- "不仅……而且……" → 拆分或简化
- "与此同时" → "同时"或删除
- "总而言之" / "综上所述" → 删除
- "不可否认的是" → 删除，直接写事实
- "值得注意的是" → 删除，直接写事实
- "某种程度上" / "某种意义上" → 删除

**2. 心理锚点**（来源：novelist 句法红线6#）：
- "X知道" → 直接写后面的内容
- "X感觉到" → 直接写后面的内容
- "X意识到" → 直接写后面的内容
- "X想起" → 仅当是刻意安排的回忆时保留，否则删除
- "X不禁" → 直接写动作
- "X仿佛" → 直接写比喻的本体

**3. 冗余修饰**：
- "极致的" / "无比的" / "彻底的" / "疯狂地" / "剧烈地" → 除非真正达到了极致，否则删除修饰词
- "非常" / "十分" / "极其" → 删除，让被修饰的词自己说话
- 双重形容词 → 保留最精确的一个（"冰冷刺骨的寒风"→"刺骨的寒风"）

**4. 形容词压缩**（来源：novelist — 1名词最多1形容词）：
- "那是一个古老而破败的小屋，在暮色中投下长长的阴影" → "那是一间破败的小屋。暮色把它拉成一道细长的影子。"
- 规则：如果两个形容词修饰同一个名词 → 删除较弱的那个
- 例外：如果两个形容词各自承担不同的感官功能（如"冰冷刺骨"=触觉+视觉）→可保留

**5. 否定句转肯定句**（来源：novelist 句法红线2# 增强）：
- "他并非不知道这样做的后果" → "他知道后果"
- "不可否认的是" → 删除，直接写事实
- "不是没有可能" → "有可能"
- "并非不可能" → "可能"
- 规则：任何双重否定 → 转为肯定。任何"不可否认/毫无疑问"等前置 → 删除后直接写
- 例外：角色对话中的否定句保留——这是角色说话方式的特征
- 补充规则：叙述中的单个"没有"用于否定状态时→改为描述客观存在。例："他没有说话"→"他沉默着"

**6. 具象化检查**（来源：openclaw 具象化三原则 — 润色阶段二次验证）：
- 搜索"过了一会儿|不久之后|一段时间后" → 改写为可感知的时间流逝
- 搜索"一群|很多人|不少人" → 计数或具象化
- 搜索"大量|巨额|极强|非常|十分|极其" → 替换为可感知后果或精确数字
- 例外：角色对话中允许模糊表达。叙述中必须具象化

**7. 形而上学总结**（来源：novelist 句法红线4#）：
- "命运的齿轮……" → 删除
- "历史的洪流……" → 删除
- "无人知晓的角落……" → 删除
- "冥冥之中……" → 删除
- 任何对事件意义的抽象评价 → 删除，让读者自己感受

### Priority 2：句子节奏优化（逐句自检方法 — 来源：novelist）

**1. 长短句交替检查**：
- 逐句标注：短句(5-15字) / 中句(16-25字) / 长句(26-40字) / 超长句(>40字)
- 连续3句以上长度相同（都长/都短）→ 调整其中一句的长度
- 动作场景：短句为主（5-15字）——如果出现25字以上的长句→拆分
- 描写/内心场景：允许长句——但如果一段全是长句→插入一句短句打破节奏
- 超长句(>40字)：除非是有意为之的蓄力句，否则必须拆分

**2. 段落形态检查**（来源：openclaw）：
- 单句段比例应保持在30-50%
- 如果某段结尾可以独立成句且具有冲击力→提取为单句段
- 连续3段长度相同 → 调整其中一段的长度

**3. 破折号检查**：
- 动作场景中的"——"接力 → 改为句号
- 但保留有功能的破折号（突然转折/打断/强调式解释）
- 如果一段中破折号≥3个 → 至少将其中1个改为句号

**4. 句首多样性**：
- 提取每句话的前4个字符 → 标记相同模式
- 如果连续3句以相同主语开头（"他"重复）→ 重组其中一句的句首
- 改变方式：时间状语前置/地点状语前置/动词开头/被动结构

**5. 西式从句拆分**（来源：novelist 句法红线5#）：
- 搜索"当……的时候"长句 → 拆分为独立短句
- 定语>15字的句子 → 将定语拆成独立句子
- "的"字连续出现≥3次 → 重组句式
- 例："当他站在那个曾经见证无数修士来来去去如今已荒废的广场上时" → "他站在广场上。这里曾见证无数修士的来去。如今荒废了。"

### Priority 3：对话自然度（来源：novelist 对话排毒 + openclaw）

**1. 去书面语腔**：
- "为何" → "为什么"（除非角色设定为古风雅言）
- "已然" → "已经"
- "可否" → "能不能" / "可以吗"
- "如此" → "这么" / "这样"
- "罢了" → 保留（这是有功能的语气词）

**2. 对话排毒**（来源：novelist）：
- 删除对话中的AI腔倒装——书面语引用 → 改为口语
- 对话中避免"既然……那么……""虽然……但是……"的完整书面语复句结构
- 角色对话中允许不完整句——真实的人说话是碎的、跳的、吞回去的
- 例："我本来想……算了，你不会明白的。" > "我本来想跟你说，但是你不可能明白这件事。"
- 规则：对话越长越礼貌 → 越短越真实。高情绪场景中，对话应该缩短

**3. 对话标签优化**：
- "说道" → 如果有更精确的动作/语气→替换。如果没有→保留（"说"不是禁用词）
- 避免连续对话全部用"X说""Y说"——穿插动作替代
- 例："他说：'走吧。'她回答：'好。'" → "'走吧。'他站起身。'好。'"

**4. 对话与叙事的穿插**：
- 连续对话超过8轮无叙事穿插 → 在合适位置插入动作/环境/微表情
- 每3-4轮对话至少有一个非对话行（动作/环境/内心反应）

**5. 角色语言指纹保护**：
- 对照角色档案中的语言指纹
- 不要"统一"不同角色的说话方式——他们应该各有腔调
- 如果发现某个角色的对话被润色后"变味了"→回退

### Priority 4：感官描写强化

1. 统计当前感官覆盖（视觉/听觉/触觉/嗅觉/味觉）
2. 如果覆盖感官 < 3种 → 在合适段落中补充缺失的感官。优先补充触觉——最被忽视的沉浸工具。每处补充不超过20字。补充位置：环境描写段或动作段。补充方式：不是加形容词——是加感官层，不是"很冷"而是"指尖发麻的冷"
3. 如果某段是连续3段纯视觉 → 在其中一段补充其他感官
4. 如果本章的感官锚点（从 Memo 读取）被淡化 → 强化该感官的描写

### Priority 5：标点与格式统一

1. 标点检查：中文引号统一、省略号…（U+2026）、破折号—（U+2014）、句号不在行首
2. 段落间距：段落之间空一行。对话行与叙事行之间不额外加空行
3. 数字与单位：中文数字用于叙述（"三天后"保留，不改为"3天后"）。按 Genre Profile 配置

## 三、Step 3 — 润色后自检

```
□ 字数变化：润色后与润色前对比，差值 ≤ ±30字？超出→回退

□ AI腔检查：搜索套话模式列表→是否已全部清除？
□ 心理锚点：搜索"知道|感觉到|意识到|不禁|仿佛|宛如"→清空或改写？
□ 否定句转换：搜索"并非不|不可否认|不是没有|并非不可能"→转为肯定句？
□ 形容词压缩：每段→是否有1个名词被2个以上形容词修饰？
□ 具象化：搜索"过了一会儿|一群|大量|巨额|极强"→已具象化？
□ 形而上学：搜索"命运的|历史的|无人知晓|冥冥"→已清空？
□ 违禁词：按 Genre Profile 违禁词表逐一检查→润色过程中无意引入了？
□ 疲劳词：按 Genre Profile 疲劳词表逐一统计→有词频超标？

□ 高峰体验句保护：PROTECTED句子未被大幅修改？
□ 作者说区域：完全未触碰？
□ 隐藏情节追踪卡：完全未触碰？
□ 章末钩子：最后2段钩子力度保持不变？
□ 角色对话：各角色保留了各自语言特点？

□ 风格指纹对照：润色后句长分布/感官分布与润色前对比→偏差>15%？有意为之→可接受。非有意→回退

□ 润色后全文通读一遍→有"拼接感"或"过度润色"？有→回退。好润色是"读不出润色痕迹"
```

## 五、校准与进化

### 润色有效性追踪

- □ 哪些 AI腔模式最常在润色中被移除 → 反馈给 Writer PRE_WRITE_CHECK
- □ 哪些心理锚点词最常残留到润色阶段 → 建议 Writer 升级该词警觉级别
- □ 哪些感官最常被补充 → 反馈给 Planner——在 Memo 中更强调该感官锚点
- □ 哪些润色操作被 Auditor 后续标记为"润色引入了新问题" → 调整润色策略

### 风格指纹校准（来源：onkos style_learner 6 维框架）


- □ 句式：avg_length 变化？润色后 short_pct 显著上升→可能过度拆分句子。pattern 从 concise 变 elaborate→可能引入了冗余修饰
- □ 词汇：richness 下降 > 10% → 可能过度替换了多样性词汇。genre_markers 被抹除→过度去 AI 化丢失了题材特征
- □ 修辞：metaphor_count 变化？润色后比喻全部消失→可能过度执行了"少用比喻"红线。爆发章允许适度保留
- □ 情感：polarity 是否保持？极性反转→润色过度改动了情感表达
- □ 节奏：rhythm_variety 下降 → 段落长度被统一化→节奏单调
- □ 对话：ratio 下降→可能删除了部分对话。tag_diversity 被统一→角色对话标签单一化

偏差阈值：单维度 < 15% → 正常。15-30% → WARNING。>30% → CRITICAL（过度润色——回退该维度润色操作）

### 润色策略优化（积累≥20章润色数据后）

- 某类操作始终高频→问题源头在 Writer 生成阶段
- 某类操作始终低频→该操作可能不够必要→降低优先级
- 润色后字数变化频繁接近 ±30 上限→润色可能过深→更严格遵循"最小改动"原则
