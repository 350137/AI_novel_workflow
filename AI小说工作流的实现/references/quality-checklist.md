# 质量自查清单

> 基于 tomato-novelist 质量检查清单 + novelist 句法红线 + openclaw 反AI写作。
> Writer 在每章正文写完后、提交审计前使用。

---

## 一、基础要素

- [ ] 章节有明确标题
- [ ] 字数 2200-2800（正文，不含作者说）
- [ ] 包含作者说 + 隐藏情节追踪卡
- [ ] 时间地点清晰

---

## 二、开头检查（前20%决定生死）

- [ ] 前3段内抓住读者（有行动/冲突/悬念）
- [ ] 不是天气或日常流程开场
- [ ] 与上一章有连接（承接钩子或明确跳跃）
- [ ] 背景信息不过量（无大段信息倾倒）

---

## 三、句法红线（novelist 6条，零容忍）

- [ ] 红线1：比喻数 ≤ 3个/章
- [ ] 红线2：无双重否定（"并非不/不可否认/不是没有"）
- [ ] 红线3：动作场景无破折号接力（"——"→"。"）
- [ ] 红线4：无形而上学总结（"命运的齿轮/历史的洪流/无人知晓/冥冥之中"）
- [ ] 红线5：无西式从句堆叠（"当……的时候"长句→拆分）
- [ ] 红线6：无心理锚点（"他知道/感觉到/意识到/不禁/仿佛/宛如"）

---

## 四、反AI写作（openclaw v4）

- [ ] 30-50% 段落为单句段
- [ ] 无连续3段长度相同
- [ ] 长短句交替，无连续5句长度相同
- [ ] 无套话（"在……的过程中/与此同时/总而言之/值得注意的是"）
- [ ] 无冗余修饰（"极致的/无比的/彻底的/疯狂地/剧烈地"）
- [ ] "的"字不密集（无连续3个"的"）

---

## 五、具象化三原则

- [ ] 时间具象化：无"过了一会儿/不久之后/一段时间后"→ 改为可感知的时间流逝
- [ ] 人物具象化：无"一群/很多人"→ 计数或具象化
- [ ] 尺度具象化：无"大量/巨额/极强/非常/十分/极其"→ 替换为可感知后果

---

## 六、对话检查

- [ ] 每句对话有目的（推进情节/揭示人物/制造冲突）
- [ ] 不同角色说话腔调不同（不统一）
- [ ] 对话中有潜台词（不是所有话都直说）
- [ ] 对话标签不重复（"说"重复率<30%）
- [ ] 高情绪场景中对话缩短

---

## 七、感官检查

- [ ] 覆盖 ≥3 种感官
- [ ] 无连续3段纯视觉描写
- [ ] 触觉有至少1处（最被忽视的沉浸工具）

---

## 八、钩子检查

- [ ] 章末有钩子（过渡章可豁免）
- [ ] 钩子类型与 Memo 一致
- [ ] 钩子强度与章节类型匹配
- [ ] 章末句式/意象与近3章结尾不重复
- [ ] 不是虚假悬念或低风险钩子

---
