# 剧情结构

> 来源：novelist Zone 2 + inkos Architect + story-writer + tomato-novelist
> 使用方：Advisor (Phase 2-S1/S1B/S2/S5) / Planner

---

## 一、故事结构模型

三种模型供用户选择，各题材有默认倾向：

| 模型 | 结构 | 适用 | 题材默认 |
|------|------|------|---------|
| **爽文模型** | Goal → Obstacle → Cost → Reversal（逆转突破） | 快节奏打脸文 | 玄幻 |
| **传统模型** | Setup → Development → Twist → Resolution（收束） | 有深度的成长故事 | 悬疑/种田 |
| **混合模型** | 日常/过渡用传统，高潮/打脸切换爽文 | 节奏灵活的叙事 | 都市 |

**各题材默认结构**：玄幻→爽文；悬疑→传统（严格）；都市→混合；种田→传统。用户可覆盖默认值。

## 二、主题与基调（story_frame 段1示例）

**写命题的方法**：
不是"从弱到强"的空话。是可用一章验证的陈述。

**好命题**："'废材'标签是社会用来控制人的工具——主角用三年苦功证明了这一点"
**差命题**："这是一个关于成长的爽文"

**基调的写法**：
不是"热血"。是具体的感官-情绪对位描述。
**好基调**："外冷内热——日常描写的自然环境是冷的，但主角的意志和导师的藏锋是热的。这种温差本身营造张力"

**目标读者画像**：
不是"男频读者"。是"喜欢扎实修炼细节、反感无脑开挂的25-35岁类型读者"

**黄金开篇3版本**：
- 版本A（强冲突型）：直接从事件切入，一句话包含冲突
- 版本B（强反差型）：突出隐藏身份或能力与处境的矛盾
- 版本C（强情绪型）：用极致情感一秒建立情绪锚

## 二、三幕式结构 + Save the Cat 15 节拍

全书分三幕：第一幕铺垫(~25%)、第二幕对抗(~50%)、第三幕解决(~25%)。

长篇(300+章)→第一幕=1-2卷 / 第二幕=3-6卷 / 第三幕=1-2卷。

**Save the Cat 15 节拍（★ = 必须标注的5个关键节拍）**：

```
第一幕 — 铺垫（~25%）
  1. 开场画面：主角的日常——展示世界观和主角初始状态
  2. 主题陈述：暗示故事核心主题
  3. 铺垫：介绍配角、世界观细节
★ 4. 催化剂：突发事件——打破日常平衡
  5. 争论：主角犹豫是否接受挑战
★ 6. 第二幕节点：主角做出决定——踏上无法回头的旅程

第二幕 — 对抗（~50%）
  7. B故事：新的关系线（爱情/导师/盟友）
  8. 游戏时间：探索新世界，小试牛刀
★ 9. 中点：重大转折——"赢"的定义发生变化。伪胜利或伪失败。
  10. 坏人逼近：内外压力增大
★ 11. 一无所有：最黑暗时刻——主角失去了什么？
  12. 灵魂黑夜：主角反思——找到新方向

第三幕 — 结局（~25%）
  13. 第三幕节点：集结力量/知识/盟友
★ 14. 高潮：最终对决——核心信念正面碰撞
  15. 终场画面：展示主角新生活——呼应开场画面
```

## 三、前台/后台双层冲突

来源：inkos Architect。

**前台故事**（读者每章看到的表层冲突）：写清楚"冲突的齿轮如何咬合"。不是事件列表。

**示例**："主角需要在以境界论尊卑的微型江湖中存活，同时暗中恢复实力——每一步都面临暴露与忍耐的冲突：公开实力则引来掌权者的扼杀，隐藏实力则在宗门任务中处处受制。"

**后台故事**（贯穿全书的暗线机器）：前台事件为什么发生的深层原因。不是"更大的Boss"。

**示例**："金手指的封印不是意外——掌权者当年参与了对主角家族的清洗，金手指被封印在目标对象体内作为'活体容器'。掌权者不是在打压一个天才，而是在看守一座监狱。主角的每一次突破都在逼近这个真相。"

**双层因果关联**：每段前台冲突必须能追溯到后台齿轮——不是"后面会有关联"，而是写清楚具体机制："第X章的前台事件→揭露了后台的Y线索"。写不出具体机制 → 后台暗线还不够具体。

## 四、多版本结局设计（4种）

来源：novel-writer-cn。必须在创意阶段规划多种结局——不是为了"选一个写"，而是为了：(1) 让伏笔设计足够灵活 (2) 让人类在创作中途可以切换结局方向 (3) 增加故事的不可预测性

| 类型 | 描述 | 闭合度 |
|------|------|--------|
| **A. 经典圆满** | 主角达成目标，主要冲突解决 | 高 |
| **B. 悲剧/代价** | 主角付出代价，目标部分达成或完全失败 | 高 |
| **C. 开放/悬念** | 核心冲突解决但打开新维度，留续作空间 | 低 |
| **D. 反转/身份揭示** | 出人意料，角色身份揭示或主题升华 | 中 |

4种结局都必须与 story_frame 兼容——同一个故事的4种可能收束方式。

**结局与篇幅的关系**：短篇(<100章)→通常走向经典圆满或悲剧结局（闭合性强）；长篇(300+章)→开放结局或反转结局更合适（有余地展开续作）。

**全书Objective 示例**："主角恢复全部被封印的天赋，揭露封印背后的真相，并做出一个定义'强者责任'的选择——守护或摧毁这个封印体系。"（Objective 不是情节概述——是"对主角意味着什么"的陈述）

## 五、卷级节奏设计

来源：inkos Architect + tomato-novelist。

### 分卷要素（每卷）
- 卷主题（本卷探讨什么，不是事件概括）
- 卷情绪曲线（从什么情绪→经历什么→到达什么情绪）
- 卷 OKR（Objective + 3 Key Results）— KR 必须是可验证状态，每3-5章推进一个 KR
- 卷间钩子（前台钩子 + 后台钩子，双层覆盖）
- 不可逆事件（≥1件，信息/关系/物理/权力改变）

**卷主题示例格式**：
"第一卷'<卷标题>'（第1-N章）——探讨'本卷核心命题'。情绪从<初始情绪>开始（<具象描写>），经过<中间阶段>，在<高潮事件>时达到爆发——然后立刻被<后续危机>拉回现实。卷尾状态：主角从<卷首状态>变为<卷末状态>。"

**OKR 示例格式**：
```
O： 主角从"被封印的X"变为"暗中恢复实力的隐藏威胁"
KR1：能力基础修复至正常水平 ✓
KR2：金手指第一层法则被激活，核心封印全部碎裂 ✓
KR3：在危机中首次公开展露实力，引发至少一名势力高层的警觉 ✓
```
注意：KR 不是"发生什么事"而是"达成什么状态"。

### 节奏6原则（至少3条具体化到本书）
1. [必选] 每卷有1个核心爽点高潮 + 1个情绪转折点
2. [必选] 连续3章无冲突推进 → 节奏失速警告（Auditor标记）
3. [可选] 高潮前3-5章必须埋至少1条新伏笔
4. [可选] 高潮后1-2章必须写"改变后的世界"
5. [可选] 卷与卷之间必须有"状态重置但威胁升级"
6. [可选] 每卷第1章必须建立核心问题，卷末必须回答（引出更大问题）

## 六、伏笔12列体系

来源：inkos Phase 7。初始化：≥15条初始伏笔，其中 3-7 条 core_hook=true。

| 字段 | 说明 |
|------|------|
| hook_id | H001-H999 |
| description | 伏笔内容 |
| type | character_secret / world_truth / power_foreshadow / relationship_foreshadow / threat_foreshadow / item_mystery |
| priority | 5=核心主线必回收 → 1=氛围性可选回收 |
| core_hook | true/false — 全书仅3-7条核心伏笔 |
| depends_on | 依赖的其他伏笔ID列表 |
| planted_chapter | 埋设章 |
| expected_payoff | 预期回收章（不能填 open） |
| payoff_timing | exact(精确章号) / range(范围) / era(卷级) / open(待定) |
| half_life | 超过此章数未推进→urgency衰减。核心伏笔长(20-30章)，次要短(5-10章) |
| notes | 备注 |

## 七、英雄之旅 12 阶段（补充结构模型）

来源：plot-structures.md。与 Save the Cat 互补——长篇史诗优先用英雄之旅，快节奏爽文优先用 Save the Cat。

| # | 阶段 | 说明 | 对应三幕 |
|:--:|------|------|:--:|
| 1 | 平凡世界 | 主角的日常——展示缺陷和缺失 | 第一幕 |
| 2 | 冒险召唤 | 打破日常的事件——不可忽视 | 第一幕 |
| 3 | 拒绝召唤 | 主角犹豫/恐惧/找借口 | 第一幕 |
| 4 | 遇见导师 | 引路人出现——提供工具/知识/信心 | 第一幕 |
| 5 | 跨越门槛 | 主角做出决定——进入新世界，不可回头 | 第一幕→第二幕 |
| 6 | 考验·盟友·敌人 | 新世界的试炼——结识盟友，识别敌人 | 第二幕 |
| 7 | 接近核心 | 逼近最大的挑战——准备最终对决 | 第二幕 |
| 8 | 磨难/至暗时刻 | 最大的考验——主角面对死亡/最大的恐惧 | 第二幕 |
| 9 | 获得奖赏 | 度过磨难后获得——力量/知识/物品/盟友 | 第二幕→第三幕 |
| 10 | 返回之路 | 带着奖赏回归——但回归路上有新挑战 | 第三幕 |
| 11 | 复活/最终对决 | 终极考验——应用所有学到的教训 | 第三幕 |
| 12 | 带着灵药归来 | 主角回归平凡世界——但已改变，世界也因此改变 | 第三幕 |

**英雄之旅 vs Save the Cat**：

| 维度 | 英雄之旅 | Save the Cat |
|------|---------|-------------|
| 适用 | 史诗/成长/冒险 | 所有类型 |
| 主角 | 从平凡到非凡 | 多种弧光可能 |
| 理念 | 向内探索→向外改变世界 | 结构节拍→商业类型 |
| 互补 | 长篇史诗优先用英雄之旅；快节奏爽文优先用 Save the Cat |

## 八、悬疑 7 阶段结构

来源：plot-structures.md。

| # | 阶段 | 说明 |
|:--:|------|------|
| 1 | 谜题/案件引入 | 核心谜团呈现——读者知道"要解什么谜"。同时埋下红鲱鱼 |
| 2 | 初步调查 + 红鲱鱼 | 主角展开调查——收集线索。读者被红鲱鱼引导到错误方向 |
| 3 | 首次突破线索 | 第一条真正指向真相的线索出现——但尚未拼成完整画面 |
| 4 | 复杂化 | 新受害者/新嫌疑人/新证据——看似推翻之前的推理 |
| 5 | 关键揭示 | 一条线索改变了整个图景——真相开始浮现 |
| 6 | 最终推理 + 对峙 | 主角拼出完整真相——与凶手/真相正面碰撞 |
| 7 | 解决 + 揭示 | 真相大白——回顾所有线索如何指向真相 |

**悬疑结构特殊规则**：
- 红鲱鱼只在阶段 2-3 有效——阶段 5 之后引入的新误导线索 = 不公平
- 阶段 5 的关键揭示必须在阶段 1-2 就埋下了线索（公平游戏）
- 阶段 7 必须回答"为什么是这个人/这个真相"——不只是"是谁"而是"为什么"

## 九、言情 7 阶段结构

来源：plot-structures.md。

| # | 阶段 | 说明 |
|:--:|------|------|
| 1 | 初次相遇 | 令人印象深刻的第一次见面——不一定是正面的 |
| 2 | 吸引 + 抗拒 | 双方开始注意到对方——但各自有理由抗拒 |
| 3 | 逐渐靠近 | 被迫共处/合作——强迫接近创造了解机会 |
| 4 | 中点确认 | 情感被确认——紧接着被重大冲突打断 |
| 5 | 重大冲突/分手 | 隐藏的秘密/误解/外部障碍爆发 |
| 6 | 关键姿态/和解 | 一方做出重大姿态证明爱情——不是"说"而是"做" |
| 7 | 幸福结局(HEA) | 双方克服障碍——情感满足收束 |

**言情结构特殊规则**：
- 阶段 1 必须在全书前 10-15% 内完成
- 阶段 3 是甜度最高的区域——适当延长但不超过全书 30%
- 阶段 5 的分手原因必须可理解——不能是"因为误会"拖 20 章
- 阶段 6 的姿态必须具体——牺牲/选择/改变

## 十、钩子技法摘要

来源：hook-techniques.md。Volume_map 设计卷间钩子时的参考。

**13 种章末悬念钩子**：悬念钩子 / 威胁钩子 / 逆袭钩子 / 情感钩子 / 揭示钩子 / 两难钩子 / 时间锁钩子 / 身份钩子 / 反套路钩子 / 代价钩子 / 世界钩子 / 共鸣钩子 / 误导钩子

**钩子强度分级（5 级）**：
- 1-2 级（弱钩子）：日常章/过渡章使用——氛围悬念/小期待
- 3 级（中钩子）：蓄压章使用——明确威胁/即将发生的事
- 4-5 级（强钩子）：爆发章/卷末章使用——生死抉择/重大揭秘

**三段式钩子编排**：第 N 章末埋钩（播种）→ 第 N+1 章承接（生长）→ 第 N+X 章回收（收获）

**钩子禁忌**：虚假悬念（用了读者会弃书）/ 机械降神（Boss 突然变弱）/ 过度留白（连续 3 章不收钩）/ 低风险钩子（读者不在乎结果）
