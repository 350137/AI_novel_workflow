# 人物设计

> 来源：fiction-crafter + inkos Architect + novelist Zone 1 + onkos CharacterMind
> 使用方：Advisor (Phase 2-S3) / Planner / Writer / Auditor (OOC检测)

---

## 一、角色反差公式

每个角色 = 核心标签 + 反差细节 = 立体人物。

**传播力标准**：一句话读给陌生人听，对方3秒内不想追问→重写。

**好例子**：
- 主角：被封印的废材 + 三年从未停止修炼 = "不是因为天才才努力，而是努力到别人以为他是天才"
- 导师：寄居在金手指中的远古残魂 + 说话刻薄但从未真的让主角去送死 = "他骂你是废物，但每次你撑不住的时候他第一个出手撑住你"
- 盟友：宗门的天才剑修 + 对宗门体制有一种自己也说不清的厌恶 = "她赢过所有人，但赢的感觉越来越像在完成别人的任务"

**坏例子（禁止）**：
- "冷酷杀手但内心温柔"（太泛，没有具体细节）
- "天才但懒惰"（陈词滥调）
- "强大但不爱说话"（不是反差，是常见设定）

## 二、角色 8 段档案格式

每个主要角色（≥3个）独立文件，8段结构：

1. **核心标签**（1行）— 一句话传播标签（见上方反差公式）
2. **人物小传**（1段散文，100-200字）— 因果叙事：因为___，所以他变成了___样的人
3. **性格剖面**（三维度）— 表面性格 / 深层性格 / 性格弱点（不能是"太善良"这种伪缺陷）
4. **Big Five 人格量化**（详见下方第三节）
5. **禁止行为**（OOC 红线，≥3条）— Auditor 用于字符串匹配 + 语义近似检测
6. **核心驱动力** — 表层目标 / 深层渴望 / 对立面 / 改变触发条件
7. **成长弧线** — 起点状态 → 转折点 → 终点状态 → 成长的代价（不是白送的）
8. **关系网络** + **语言指纹** — 与其他角色的关系本质 / 口头禅 / 说话方式

**次要角色 3-5 个**：简化版，只需核心标签+反差细节+1段小传。

## 三、Big Five 人格量化

来源：onkos CharacterMind。用于 Auditor OOC 检测时量化行为偏差。每个维度 0.0-1.0 打分。

| 维度 | 低分特征 (0.0-0.3) | 高分特征 (0.7-1.0) |
|------|-------------------|-------------------|
| **开放性(O)** — 保守务实↔开放创新 | 不轻易接受新观念、依赖经验、排斥变化 | 好奇探索、乐于尝试新方式、容易被新鲜事物吸引 |
| **尽责性(C)** — 随性自由↔严谨自律 | 随性、拖延、不守规矩 | 极度自律、信守承诺、做事有计划 |
| **外向性(E)** — 内敛沉静↔外向热情 | 话少、独处优先、社交耗能 | 主动社交、善于表达、人群中精力充沛 |
| **宜人性(A)** — 独立强势↔温和合作 | 竞争性强、不太在意他人评价、可能显得冷酷 | 善解人意、优先合作而非对抗、在意他人感受 |
| **神经质(N)** — 情绪稳定↔敏感多虑 | 情绪稳定、抗压能力强、不易动摇 | 敏感内耗、情绪波动大、易焦虑 |

**OOC 检测原理**：某章中角色行为偏离其 Big Five 评分 >0.5 → 标记 OOC 风险。例：E值0.2的角色（内敛）某章在众人面前主动长篇大论 → 标记。

**人格一致性**：Big Five 描述必须与散文描述一致。例：O:0.3 C:0.9 E:0.2 A:0.4 N:0.3 → 与"被嘲讽三年面不改色、独自苦修、话少但每句有分量"一致。

## 四、成长弧线模型

来源：novelist + onkos CharacterMind。

**弧线结构**：
- 起点状态（卷首的角色是什么样？）
- 中间转折点（什么事/什么人触发了成长方向的变化？）
- 终点状态（卷末的角色变成了什么样？）
- 成长的代价（成长不是白送的——失去了什么/放弃了什么/变成了什么他曾经不是的人？）

**代价示例**："他不再能只是'那个在小屋里独自治愈自己的人'。重塑能力之后，他必须面对世界——而这意味着一场他从未准备好的战争。"

**弧光类型**（来源：character-arc-guide.md）：
- 正向弧（缺陷→成长→圆满）— 最常见
- 负向弧（堕落/黑化）— 反派或悲剧主角
- 平坦弧（坚守信念，改变世界而非被世界改变）— 配角或特定主角类型

**弧光六阶段（标准化模型）**：

| 阶段 | 名称 | 描述 | 检测信号 |
|:--:|------|------|---------|
| 1 | **缺陷** | 角色内心的弱点/错误信念 | 面对选择时反复做出"错误"决定 |
| 2 | **需求** | 角色真正需要但不知道的东西 | 因为缺陷失去了什么 |
| 3 | **抗拒** | 角色不愿改变，坚持旧模式 | 机会出现时拒绝/逃避/找借口 |
| 4 | **觉醒** | 关键事件触发反思 | 第一次承认"我可能错了" |
| 5 | **成长** | 角色开始尝试新行为模式 | 面对类似选择时做出不同决定 |
| 6 | **圆满** | 角色克服缺陷，获得成长 | 在终极冲突中应用新认知 |

**各阶段最小篇幅**：

| 阶段 | 短篇（<100章） | 中篇（100-300章） | 长篇（300+章） |
|:--:|:--:|:--:|:--:|
| 缺陷+需求 | 1-3章 | 3-10章 | 5-15章 |
| 抗拒 | 3-5章 | 10-20章 | 20-40章 |
| 觉醒 | 1-3章 | 3-5章 | 5-10章 |
| 成长 | 5-10章 | 20-40章 | 50-100章 |
| 圆满 | 1-3章 | 3-10章 | 10-20章 |

**弧光里程碑标记**（Planner 在卷级大纲中应标注）：
```
角色X 弧光里程碑：
  缺陷展示：第3-5章（连续3次拒绝他人帮助）
  需求暗示：第8章（因独自扛失去重要机会）
  抗拒高峰：第12-15章（面对团队邀请，选择独自行动）
  觉醒事件：第18章（独自行动导致队友受伤——认识到"一个人不够"）
  成长初现：第20-25章（第一次主动寻求帮助）
  圆满验证：第48-50章（卷末大战中依靠团队取胜）
```

**弧光设计原则**：
1. 弧光起点必须在卷首 5 章内通过至少一个具体行为展示
2. 转变必须有外部事件触发——不能突然"想通了"，必须有催化剂
3. 每个主要角色有独立的弧线——配角不是为了衬托主角而存在
4. **弧光与情节互为因果**：不是"角色成长了然后剧情变化"——而是"剧情逼迫角色成长"
5. **弧光回弹是允许的**：角色在成长后可能因新事件回退到旧模式——这是真实的人物塑造。但必须有合理的触发事件

**改变触发条件**：不是"经历了什么"，而是"他内心必须接受什么之前不愿接受的事实"。例：只有当主角亲眼看到他信任的人在战斗中因他而受伤——他才可能放下"我一个人就够了"的信念。这被用来验证：故事中是否有足够的事件来推动这个改变？

**弧光停滞双重检测**：
- 粒度变化检测：连续 5 章无能力/地位/认知可感知变化 → Auditor 标记 WARNING
- 弧光阶段检测：角色在 ≥15 章内未展示弧光阶段推进 → Auditor 标记 WARNING（弧光停滞）
- 两者是互补检查——粒度检测捕捉"原地踏步"，阶段检测捕捉"重复同类变化而无实质推进"

**核心驱动力示例**：
```
表层目标："恢复实力，不再被看不起"
深层渴望："证明废材标签是谎言，三年苦功没有白费"
对立面："整个势力以境界论人的体制——恢复实力本身无法推翻体制"
```

**内在驱动示例**（1段散文）：驱动他行动的内在逻辑——不是"他想要变强"，而是"他认为只要够强就能___，这个信念来自___的经历，但这个信念将在___时受到挑战"。

**语言指纹示例**："话不多，但每句话都带着三年被鄙夷磨出的冷。不会说'我一定会变强'——会说'我已经在变强了，你们只是还不知道。'"

## 五、配角功能定位

来源：character-template-tomato.md。

| 类型 | 功能 | 示例 |
|------|------|------|
| 导师型 | 传授技能/知识，引导主角成长 | 师父、前辈、引路人 |
| 盟友型 | 并肩作战，提供助力 | 同伴、搭档、战友 |
| 搞笑型 | 调节气氛，缓解紧张 | 话痨、活宝、吐槽担当 |
| 爱情型 | 情感线核心，推动主角情感成长 | 恋人、暗恋对象 |
| 叛徒型 | 表面友好实则对立，制造反转 | 内奸、卧底、背叛者 |

### 人物关系张力图

来源：character-template-tomato.md。

| 关系对 | 初期状态 | 中期转折点 | 预期终点 | 戏剧潜力 |
|--------|---------|-----------|---------|---------|
| 主角↔导师 | 利用/试探 | 导师暴露真实目的 | 真正的师徒/分道扬镳 | 高——信任与背叛 |
| 主角↔同伴 | 互不信任 | 生死关头互相救命 | 生死之交 | 中——渐进的信任 |
| 主角↔反派 | 敌对 | 发现反派动机的合理性 | 惺惺相惜/你死我活 | 高——价值观碰撞 |

### 出场设计

来源：character-template-tomato.md。每个主要角色在首次出场时明确：

- **首次出场方式**：行动中（展示核心能力）/ 冲突中（展示核心性格）/ 误解中（制造戏剧张力）
- **首次出场要展示什么**：核心标签（第1秒认知）/ 反差细节（让角色立体）/ 核心欲望暗示（让读者知道他要什么）
- **创伤经历/关键记忆**：在人物小传中明确标注——什么过去塑造了他现在的性格弱点
