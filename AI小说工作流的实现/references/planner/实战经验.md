## 一、核心教训

### 教训 1：不要相信"摘要版"

**问题**：初版大量章节仅 400-800 字，是无法发布的"摘要"而非完整章节。
**根因**：LLM 在缺乏详细指令时会倾向生成结构摘要而非叙事正文。
**解决**：在 Writer Prompt 中明确"2200-2500 字硬性指标"，并提供补全机制。

### 教训 2：逻辑自检比生成更重要

**问题**：生成速度快但逻辑错误率高（~30%），修复成本比重新生成更高。
**根因**：伏笔缺失、人物状态不一致、时间线混乱。
**解决**：生成前内置检查表——伏笔回收、人物状态、战力逻辑、时间线。发现断层→立即修正，而非"后面再补"。

### 教训 3：细节决定代入感

**问题**：章节"能看但不沉浸"——读者完读率低。
**根因**：缺少心理独白、环境渲染、动作细节、配角反应。
**解决**：将四类细节写入 Prompt 作为硬性要求。

### 教训 4：风格统一是基础

**问题**：章节之间风格跳跃——前一章"冷峻白描"，后一章"华丽修饰"。
**根因**：LLM 在不同会话中会改变风格偏好。
**解决**：在每次生成的 Prompt 中明确风格规范（第一人称、爽点密度、节奏把控、违禁词替换）。

### 教训 5：持续迭代，数据驱动

**问题**：凭直觉调整 Prompt 导致效果不稳定。
**根因**：缺少量化指标。
**解决**：记录每章的字数、逻辑错误率、爽点密度，基于数据调优而非凭感觉。

## 三、可复用的 Prompt 优化模式

### 3.1 字数强制控制模式

```
Prompt 中明确："正文必须严格控制在 2200-2500 字（中文）。禁止短章或注水。"
生成后检查：用 check_wordcount.py 验证
不足时补全：心理独白 + 环境渲染 + 动作细节 + 配角反应
```

### 3.2 逻辑自检清单模式

```
生成前必查：
  ✅ 伏笔是否回收？
  ✅ 人物状态是否一致？（受伤、物品、等级）
  ✅ 战力逻辑是否合理？
  ✅ 时间线是否清晰？
```

### 3.3 风格锁定模式

```
每次 Prompt 中重复：
  - 第一人称：严格保持视角
  - 爽点密度：每章至少 1 个小爽点
  - 节奏：开篇切入冲突 → 中段铺垫博弈 → 结尾留下钩子
  - 违禁词替换：杀→陨落 / 死→灰飞烟灭 / 血→猩红
```
